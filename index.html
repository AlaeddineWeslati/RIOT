RIOT OS

The Revolutionary IoT Operating System

Download this project as a .zip file Download this project as a tar.gz file

RIOT leverages a microkernel to provide an energy-efficient operating system for IoT solutions.

Developer Friendly

  • Program in C or C++ like you're used to: 
#include <stdio.h>

int main(void) {
    puts("Hello world!\n");
    while(1);
}
  • Standard development tools like gcc, gdb ...
  • Minimized hardware dependent code
  • Zero learning curve embedded programming

Efficient

  • Robustness & code-footprint flexibility due to the microkernel architecture
  • Tickless scheduler enabling maximum energy-efficiency
  • Real-time capability due to ultra-low interrupt latency (~50 clock cycles) and priority-based scheduling
  • Multi-threading with ultra-low threading overhead (<25 bytes per thread)
  • Code once, run both on 16-bit platforms (e.g. MSP430) and on 32-bit platforms (e.g. ARM)

IoT Ready

  • IoT network stack
    • 6LoWPAN
    • IPv6
    • RPL
    • TCP and UDP
  • Flexible memory management (static and dynamic memory allocation)
  • High resolution and long-term timers
  • FAT file system
  • Logging system
  • Wiselib support (C++ algorithm library, including routing, clustering, timesync, localization, security and more algorithms)

Documentation

Documentation about RIOT can be found in the corresponding Github wikis. For general issues with RIOT please refer the RIOT wiki, platform specific questions are addressed in the boards wiki. More general information about how to compile, use, and implement for RIOT are available in our doxygen documentation.

Join the RIOT!

Whether you are looking for help with writing an application for RIOT, learn more about it, or just want to stay in the loop, you are invited to join the RIOT-users mailing list. For developers who want to participate and contribute to the kernel development or integrate new MCU and platform support, the [RIOT-devel mailing list(http://lists.riot-os.org/mailman/listinfo/devel) is the right place.
